# REQ-LIB-001

## 文件資訊

- 需求 ID：`REQ-LIB-001`
- 來源情境：`SCN-LIB-001`
- 情境檔案：`docs/scenarios/SCN-LIB-001.md`
- 範圍：小型圖書櫃管理 MVP（建書、借書、還書）

---

## 1. 需求目標

建立一套可本地使用的最小可用圖書櫃管理系統，解決以下問題：

- 書籍借閱人不可追蹤
- 書籍是否可借不透明
- 借閱後長期未歸還缺乏明確狀態

---

## 2. 角色與名詞

- 管理者（使用者）：可新增書籍、執行借書與還書
- 書籍（Book）：包含書名、作者、庫存、可借數量、狀態
- 借閱紀錄（Borrow Record）：包含借閱人、借出時間、歸還時間、狀態

---

## 3. 功能需求（FR）

### FR-LIB-001 新增書籍

系統必須允許使用者新增書籍，欄位至少包含：

- 書名（必填）
- 作者（必填）
- 庫存數量（必填，正整數）

建立成功後，書籍需出現在書籍清單中。

### FR-LIB-002 書籍清單查詢

系統必須提供書籍清單，至少顯示：

- 書名
- 作者
- 庫存數量
- 可借數量
- 狀態（可借閱 / 已借出）

### FR-LIB-003 借書

系統必須允許對可借書籍執行借出操作，流程如下：

- 使用者選擇書籍並點擊借出
- 輸入借閱人姓名（必填）
- 借出成功後建立借閱紀錄

借出後系統必須：

- 可借數量減 1
- 當可借數量為 0 時，書籍狀態顯示為已借出

### FR-LIB-004 還書

系統必須允許對未歸還借閱紀錄執行歸還操作。

歸還後系統必須：

- 將借閱紀錄標記為已歸還並記錄歸還時間
- 對應書籍可借數量加 1
- 書籍可借數量大於 0 時，狀態顯示為可借閱

### FR-LIB-005 借閱紀錄可追蹤

系統必須可追蹤每筆借閱行為，至少包含：

- 書籍識別資訊
- 借閱人姓名
- 借出時間
- 歸還時間（未歸還可為空）
- 借閱狀態（BORROWED / RETURNED）

### FR-LIB-006 錯誤回應採業務錯誤碼

系統對前端回應（成功與失敗）必須包含業務錯誤碼，不可以 HTTP Status 取代。

- `00000`：成功
- `A0000`：客戶端錯誤
- `B0000`：系統錯誤
- `C0000`：第三方錯誤

---

## 4. 非功能需求（NFR）

### NFR-LIB-001 本地可執行

系統需可在本地開發環境啟動並完成核心流程（建書、借書、還書）。

### NFR-LIB-002 一致性與正確性

任一借書或還書操作完成後，書籍可借數量與借閱紀錄狀態必須一致。

### NFR-LIB-003 基本效能

在 1,000 本書籍、同時單使用者操作下，書籍清單查詢回應時間應低於 1 秒（本地環境）。

### NFR-LIB-004 可測試性（UI Locator）

UI 元件需保留穩定 `data-testid`，且後續調整不得破壞既有測試定位。

### NFR-LIB-005 可維護性

前後端命名、DTO 與 API 契約需遵循專案規範（AGENTS.md），避免硬編碼與重複邏輯。

## 5. 驗收標準（AC）

### AC-LIB-001 新增書籍成功

- Given 使用者在新增書籍表單輸入合法書名、作者、庫存
- When 點擊送出
- Then 系統新增書籍成功，清單可見該書，並顯示正確庫存與可借數量

### AC-LIB-002 借書成功

- Given 清單存在可借書籍 `Clean Code`，可借數量 >= 1
- When 使用者對該書執行借出並輸入借閱人 `Samson`
- Then 建立借閱紀錄，該書可借數量減 1，並更新狀態

### AC-LIB-003 借書失敗（無可借庫存）

- Given 書籍可借數量為 0
- When 使用者嘗試借出
- Then 系統拒絕操作並回傳對應業務錯誤碼（`A0000` 類）

### AC-LIB-004 還書成功

- Given 存在一筆未歸還借閱紀錄（借閱人 `Samson`）
- When 使用者對該紀錄執行歸還
- Then 紀錄狀態改為已歸還、填入歸還時間，且書籍可借數量加 1

### AC-LIB-005 借閱紀錄可追溯

- Given 系統已發生借書與還書行為
- When 使用者查閱借閱紀錄
- Then 可看到借閱人、借出時間、歸還時間與目前狀態

### AC-LIB-006 錯誤碼一致

- Given 任一 API 成功或失敗
- When 前端接收回應
- Then 可依業務錯誤碼判斷結果，不依賴 HTTP Status 作為唯一判斷條件

---

## 6. 範圍界線（In / Out）

### In Scope

- 單一共享書櫃管理
- 書籍新增、借出、歸還
- 借閱紀錄追蹤

### Out of Scope

- 使用者登入/權限模型
- 逾期罰則與通知
- 多據點/多書櫃同步
- 推薦系統與統計儀表板

## 8. 需求風險與待確認事項

- 目前情境未定義同一本書可同時被多人借閱時的併發規則（需於設計階段補齊）。
- 情境未定義書籍唯一鍵策略（書名可重複與否），建議 SD 階段明確定義。
- 情境未定義借閱紀錄查詢介面形式（列表、過濾條件），先以最小列表需求處理。
