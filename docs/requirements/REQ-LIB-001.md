# REQ-LIB-001

## 文件資訊

- Requirement ID: `REQ-LIB-001`
- 文件名稱: 小型圖書櫃管理需求分析
- 來源情境: `SCN-LIB-001`（`docs/scenarios/SCN-LIB-001.md`）
- 版本: `1.0.0`
- 狀態: Draft

## 1. 需求目標與範圍

本需求定義小型圖書櫃管理系統的最小可用功能（MVP），解決以下問題：

- 書籍被借走但無法追蹤借閱人
- 使用者無法快速確認書籍是否可借
- 借閱後未歸還狀態缺乏明確紀錄

MVP 功能僅涵蓋：

- 建立書籍
- 借出書籍
- 歸還書籍
- 於畫面呈現可借數量與借閱狀態

## 2. 角色定義

- 管理者（Admin）: 可新增書籍、執行借出與歸還操作、查看書籍與借閱狀態。

## 3. 功能性需求（FR）

### FR-LIB-001 建立書籍

系統必須提供新增書籍功能，欄位至少包含：

- 書名（title）
- 作者（author）
- 庫存總數（totalQuantity）

新增成功後，書籍需立即出現在書籍清單中。

### FR-LIB-002 顯示書籍清單與可借狀態

系統必須提供書籍清單，至少顯示：

- 書名
- 作者
- 庫存總數
- 可借數量（availableQuantity）
- 借閱狀態（例如：可借閱 / 已借出）

可借數量必須由「庫存總數 - 未歸還借閱數」計算。

### FR-LIB-003 借出書籍

系統必須支援從書籍清單借出指定書籍，流程至少包含：

- 選擇目標書籍
- 輸入借閱人姓名（borrowerName）
- 建立借閱紀錄（狀態為未歸還）

借出成功後，系統必須：

- 減少可借數量
- 更新書籍借閱狀態

### FR-LIB-004 歸還書籍

系統必須支援對未歸還借閱紀錄執行歸還，並於成功後：

- 將借閱紀錄更新為已歸還
- 恢復對應書籍可借數量
- 更新書籍借閱狀態

### FR-LIB-005 借閱資料一致性保護

系統必須保證以下業務規則：

- 當可借數量為 0，不可再借出
- 不可重複歸還同一筆已歸還借閱紀錄
- 所有失敗操作需回傳業務錯誤碼（不可僅依賴 HTTP status）

## 4. 非功能性需求（NFR）

### NFR-LIB-001 可用性（MVP 本地執行）

系統需可於本地環境啟動並完成新增/借出/歸還完整流程，不依賴外部第三方服務。

### NFR-LIB-002 一致性與正確性

借出與歸還後，書籍可借數量與借閱狀態更新需立即且一致，避免畫面與資料不一致。

### NFR-LIB-003 可測試性

前端可互動元件需保留穩定定位方式（`data-testid`），以支援自動化 E2E 測試。

### NFR-LIB-004 錯誤處理標準化

API 錯誤需映射至業務錯誤碼體系（`00000`/`A0000`/`B0000`/`C0000`），前端不可只以 HTTP status 判斷業務結果。

### NFR-LIB-005 UI 對齊

前端實作需對齊 `docs/figma/library-mini-admin-console/` 所提供之介面參考，且不得破壞既有 `data-testid`。

## 5. 驗收標準（AC）

### AC-LIB-001（對應 FR-LIB-001）

- Given 管理者進入系統
- When 輸入書名、作者、庫存總數後送出新增
- Then 新書籍顯示於書籍清單，且資料與輸入一致

### AC-LIB-002（對應 FR-LIB-002）

- Given 系統已有多本書籍與借閱紀錄
- When 管理者查看書籍清單
- Then 每本書皆顯示書名、作者、庫存總數、可借數量與借閱狀態，且可借數量計算正確

### AC-LIB-003（對應 FR-LIB-003）

- Given `Clean Code` 可借數量大於 0
- When 管理者對 `Clean Code` 執行借出並輸入借閱人 `Samson`
- Then 系統建立未歸還借閱紀錄，且 `Clean Code` 可借數量減少、狀態更新為已借出

### AC-LIB-004（對應 FR-LIB-004）

- Given `Samson` 對 `Clean Code` 存在未歸還借閱紀錄
- When 管理者對該筆借閱紀錄執行歸還
- Then 該借閱紀錄更新為已歸還，且 `Clean Code` 可借數量恢復、狀態更新為可借閱（若無其他未歸還紀錄）

### AC-LIB-005（對應 FR-LIB-005）

- Given 某書籍可借數量為 0
- When 管理者嘗試借出
- Then 系統拒絕借出並回傳對應業務錯誤碼

### AC-LIB-006（對應 FR-LIB-005）

- Given 某筆借閱紀錄已歸還
- When 管理者再次對同筆紀錄執行歸還
- Then 系統拒絕操作並回傳對應業務錯誤碼

## 7. 非範圍（Out of Scope）

以下不在本次 `REQ-LIB-001` MVP 範圍：

- 帳號登入與權限分級
- 預約借閱/排隊機制
- 逾期通知、罰則、催還流程
- 書籍分類標籤與進階搜尋
- 對外整合（訊息通知、第三方書目服務）
